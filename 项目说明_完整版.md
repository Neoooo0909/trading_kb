# trading_kb · A股个人投研信息处理中枢

> 把研报、聊天、短评、纪要、新闻……各类投研信息，统一**过滤提纯**成"分了成色、核了数字、可追溯、不健忘"的私人投研大脑。

# 第一部分 · 一眼看懂这是什么

## 🎯 一句话

**trading_kb 是一个投研信息处理中枢：你把各种投研信息（券商研报、群聊消息、雪球短评、调研纪要、新闻公告）一股脑丢给它，它自动从噪声里提纯出有价值的结论，标好可信度成色、核对每个数字、分门别类存档；之后你随口一问，它就给你一份带证据、带出处、带成色的投研回答。**

它做两件别人做不好的事：
- **吃进各类信息**（不挑类型，研报和群消息一视同仁地处理）
- **从噪声里提纯**（把一堆真假混杂的信息，分清成色、去伪存真、核对数字）

## 它解决的痛点

个人投研每天被海量信息淹没，真正的难题不是"信息不够"，而是——

- 📚 研报、群消息、短评看过就忘，**三个月后要用时找不到、接不上**
- 🤔 一条消息到底是**公告坐实的，还是群里的小作文**？真假混在一起分不清成色
- 🔢 AI 帮你总结，但**它会编数字**，你不敢信
- 🌊 群里短评铺天盖地，**噪声把真正的信号淹没了**
- 🔁 当初看好某个逻辑，**后来被证实还是被推翻**？没人帮你记账

trading_kb 把这些信息**统一收进来、过滤提纯、分级存档**，全自动。

## 30 秒了解：你能得到什么

| 你做什么 | 它给你什么 |
|---|---|
| 丢进各类信息（研报/聊天/短评/纪要/新闻） | 自动提纯出关键结论、因子、数字，分门别类永久存档 |
| 随口提问 | 一份"六段式"结构化回答：结论 / 证据 / 反证 / 待验证 / 交易含义 / 出处 |
| 看一条结论 | 旁边标着成色（A公告 / B研报 / C专家 / D群聊）和"几个数字核对过" |
| 关注一个标的的群里风声 | 自动聚合成情绪/热度信号，被研报印证才升级为硬证据 |
| 过段时间再问 | 被推翻的旧说法自动标记，复盘有据可查 |

## ✨ 吸引人的地方（为什么它不一样）

1. **什么信息都能吃** —— 研报、群聊、短评、纪要、新闻……不挑类型。研报走"重质量"通道精读细抽，碎片走"轻舆情"通道聚合成信号，**两条通道分流，噪声不会淹没正经证据**。
2. **从噪声里提纯，还分成色** —— 不是笼统说"利好"，而是明确告诉你这是公告级硬证据、研报级观点、还是群聊级传闻（A/B/C/D 四级），治"把小作文当买点"的病。
3. **不会编数字** —— 研报里每个数字都回原文逐一核对，AI 编的、看错的当场标红。投研里数字是命，这是别的 AI 工具给不了的。
4. **🆕 会"质疑"，不只会"总结"** —— 内置批判性体检，自动揪出三类问题：**没出处的猜测、过于乐观的数字（拿同类横向对照，年化/IC/胜率明显偏高就预警）、回测软肋（没样本外验证、区间太短）**。它会在回答里直接标红"🔴 这个数显著偏高，警惕过于乐观"。
5. **🆕 联网只采信权威** —— 需要补充信息时联网核实，但**只采信公司公告、交易所、权威财经媒体（财联社/证券时报等白名单）**，自媒体小作文一律不采信。公告直接走**巨潮资讯（cninfo，证监会指定披露平台，沪深全量、官方、免费）**，查到能把结论"坐实"升级到 A 级。
6. **永不健忘、永不假装确定** —— 三个月前的信息今天还能调出来；查不到的事老实说"待验证"，绝不编。
7. **会自动给自己打脸** —— 同一件事从"传闻"到"确认"自动更新，被推翻的旧说法标记保留，复盘时一查便知。
8. **全自动 + 本地运行** —— 你只管丢信息和提问，处理全自动；数据全在自己电脑，不上传，隐私安全。

## 怎么用（最简版）

```bash
cd ~/trading_kb
./tkb ingest                      # 把信息提纯入库
./tkb ask "多空因子 选股效果"        # 提问，得到六段式回答
```

就这么简单。

# 第二部分 · 功能模块全景（面向用户的语言）

系统由 13 个模块协作，可以理解成**两条信息通道 + 三个知识仓库 + 一个智能问答台**：研报这类长文档走"重质量通道"精读细抽，聊天/短评这类碎片走"轻舆情通道"过滤聚合。下面用大白话讲每个模块在帮你做什么。

## 🏭 摄入流水线（把各类信息提纯成知识）

### ① 研报读取器（report_lab_adapter）
**它做什么**：从你的研报处理工具 report_lab 里，把已经读好、核对过数字的研报"卡片"取出来，交给后面的流水线。
**对你的意义**：你不用重新读 PDF，它直接接住 report_lab 的成果——包括每篇研报的结论、因子、实体、已核对的数字。

### ② 分流器（classify）
**它做什么**：给每一条研报结论自动贴标签，分成四类——
- **硬事实**：订单、中标、产能、涨价这类"会兑现、能证伪"的事
- **结构关系**：谁是谁的上游、谁供货给谁这类产业链关系
- **量化事实**：因子表现、回测结果这类（你现在的研报主力是这类）
- **背景**：泛泛的"我们看好"这类，不入库只留着

**对你的意义**：不同性质的信息各归其位，查的时候不串味。

### ③ 成色评定员（grade）
**它做什么**：给每条结论评"成色"——A级公告 > B级研报 > C级专家会 > D级群聊。并且**坚持一条铁律：查不到证据 ≠ 这事是假的**，只会老实标"待验证"，绝不冤枉成"传闻"。
**对你的意义**：你一眼就知道每条结论该信几分。

### ④ 数据核查员（verify_hooks）
**它做什么**：对"订单、产能"这类可查的硬事实，去查公告、研报、行情来核实成色（默认关闭以省额度，可开启）。
**对你的意义**：把"研报说的"升级成"公告坐实的"，或者降级为"查无实据"。

### ⑤ 实体管家（entity_registry）
**它做什么**：把同一家公司的各种叫法（简称、全称、代码）归并到唯一身份证（证券代码，如 SH688017）；分清股票、基金、指数、概念，不会把"上交所"当成股票。
**对你的意义**：查"绿的谐波"和查"688017"得到同样结果，信息不会因为叫法不同而散落。

## 🔬 批判与核实（新增 · 让它会"质疑"）

### ⑭ 质疑引擎（critique）
**它做什么**：摄入每条结论时自动做"批判性体检"，揪出三类可疑——
- **没出处/猜测**：没有原文证据、数字没核对过、或满是"预计/有望/或将"却无数据支撑
- **过于乐观/夸大**：把这条的关键数字（年化、IC、ICIR、夏普、胜率…）和全库同类一比，**明显偏高（超过 90%/95% 分位）就预警**"显著偏高，警惕过于乐观"
- **回测软肋**：提到回测却没样本外/实盘验证、或回测区间太短，提醒"警惕过拟合"

**对你的意义**：它不只会总结，还会**像个挑刺的老法师告诉你"这条别全信，数字可能吹了"**。回答里直接用 🔴🟠🟡 标出来。

### ⑯ 深度质疑闭环（deep_verify）
**它做什么**：质疑模块标出可疑硬事实（如"获重大订单"）后，**自动联网拉对应公司公告正文，核对说法 vs 真实披露口径**，给三种裁决：
- 🟢 **公告佐证**：找到匹配大类（如"重大合同/中标"）的公告且内容一致 → 说法可信，升级
- 🔴 **公告打脸**：找到澄清/风险类公告且高度相关 → 说法可能不实，自动标 disputed
- 🟡 **公告无披露**：查无相关公告 → 乐观说法无公告坐实，存疑维持

**防假阳性**：核对时**扣掉公司名**（每条公告都含公司名，不算证据），只比"说法特有内容"，并要求佐证落在预期公告大类——避免拿无关公告硬凑佐证。
命令 `./tkb deep-check`（需 `TKB_USE_WEB=1`）。
**对你的意义**：从"标记存疑"进一步到"**去官方公告求证**"，研报里的乐观说法到底有没有公告坐实，一查便知。

### ⑮ 联网权威核实（web_enrich）
**它做什么**：需要补充信息时联网核实，但**只采信权威信源**，分两级——
- A 级（可坐实）：公司公告、交易所互动易、监管文件
- B 级（可佐证）：财联社、证券时报、中证报等权威财经媒体（白名单）
- 其余（自媒体/论坛/小作文）：**一律不采信，直接丢弃**

查到公告能把"研报说的"升级成"公告坐实的"（B→A）；查不到也只标"待验证"，绝不编。
**对你的意义**：信息更充分，但**只让权威信源进来，垃圾挡在门外**。

> **公告从哪来（已实测可用）**：直接走 **巨潮资讯（cninfo.com.cn）** —— 证监会指定的法定披露平台，沪深两市所有公告强制全量在此发布，**官方、最快（0.3s）、免费、无额度**，一站等于"上交所+深交所"，且**无验证码、无风控拦截**（高频会限流，已加退避重试）。
> **三项增强（对标 akshare 等高star项目同款做法）**：① **orgId 精确查询**（代码→orgId 映射，避重名公司）② **18 大类自动分类**（定期报告/业绩预告/减持/中标/重组/分红/回购/诉讼…，可按大类过滤）③ **公告正文提取**（下载 PDF + pdftotext 全文，供质疑模块核对真实披露）。
> **备用链**：巨潮拿不到时，沪市票自动兜底**上交所 query.sse.com.cn**；深市公告本就在巨潮（巨潮是深交所运营的官方平台）。
> 注：tdx 没有公告；不接 hibor 研报；智能选股有月限额 4000/月，也不用于公告。
> 实现见 `announcement.py`；默认关闭以保离线可复现，开启用 `TKB_USE_WEB=1`。

## 🗄️ 三个知识仓库（分层存放）

### ⑥ 时序事实仓（facts_store）—— 系统的核心
**它做什么**：存放所有"会变化的事实"，并且像账本一样管理它们的状态：
- **去重合并**：同一件事被多篇研报提到，合并成一条，记"被几篇支持"
- **进展替代**：从"传闻"到"意向"到"确认订单"，自动把旧的标记为已替代
- **证伪不删除**：某说法被推翻，不删掉而是标"已证伪"，复盘时还能查回

**对你的意义**：这就是你那个"不健忘、会自动打脸自己"的投研记账本。

### ⑦ 产业链结构仓（structure_store）
**它做什么**：存放"谁连着谁"的产业链关系图，支持"给我这条链上下游有哪些环节"。一条关系被几篇研报支持都记着，单篇孤证会标"低可信"。
**对你的意义**：拆解行业时，看清产业链结构。（注：需要行业研报才有料，量化研报喂不出产业链）

### ⑧ 舆情轻通道（sentiment_lane）
**它做什么**：专门处理聊天记录、短评这类碎片——先过滤掉跟你关注标的无关的，命中的标成 D级隔离起来，再聚合成"情绪/热度信号"；只有被研报印证才"升级"成正经证据。
**对你的意义**：群里的风吹草动能被捕捉成信号，但不会污染你的硬证据。

## 🔍 智能问答台

### ⑨ 六段式问答（ask）
**它做什么**：你提一个问题，它先定位到相关实体，再综合三个仓库的内容，产出固定六段的回答：**结论 / 证据链 / 分歧反证 / 后续验证 / 交易含义 / 引用来源**。每条都带成色和出处。
**对你的意义**：你得到的不是一句模糊的"看多看空"，而是一份能追根溯源、分了成色的论证。

## 🔧 幕后支撑（你不用直接打交道）

- **⑩ 配置中心（config）**：管路径和开关（用 LLM 吗、查数据吗）。
- **⑪ 数据结构（models）**：定义"结论""事实""关系""舆情"长什么样，并处理中文匹配。
- **⑫ 摄入总调度（ingest）**：把上面①~⑧串成一条龙，一条命令跑完全流程。
- **⑬ 命令行入口（cli）+ 启动器（tkb）**：你敲的 `./tkb ...` 命令就走这里。

# 第三部分 · 详细使用说明

## 一、开始之前

系统站在 **report_lab**（研报处理工具）之上：

```
你的研报 PDF → report_lab（读PDF/抽卡片/核对数字）→ trading_kb（建知识库/问答）
```

确认两点：① 电脑有 Python 3（`python3 --version` 能看到版本）；② `~/report_lab/` 里有处理好的研报（已有 63 篇）。**无需安装任何东西，开箱即用。**

## 二、最简上手：3 条命令

打开终端，进入目录（之后命令都用 `./tkb` 开头）：

```bash
cd ~/trading_kb
```

**1. 把研报装进知识库**
```bash
./tkb ingest
```
跑完显示回执（处理了几篇、抽了多少条、成色分布）。**可反复跑，不会重复入库。**

**2. 提问**
```bash
./tkb ask "多空因子 选股效果"
```

**3. 看知识库规模**
```bash
./tkb stats
```

## 三、怎么看懂回答

回答固定六段：

| 段 | 看什么 |
|---|---|
| **结论** | 一句话核心答案，带成色标签 |
| **证据链** | 支撑结论的多条证据，每条带成色+出处+数字校验数 |
| **分歧/反证** | 有没有相反观点 / 被推翻的旧说法 |
| **后续验证** | 哪些还没坐实、需要你盯着 |
| **交易含义** | 对交易的提示（低成色别当买点）|
| **引用来源** | 来自哪篇研报，可回查 |

**成色标签**（如 `[B级·待验证]`）：

| 成色 | 含义 | 怎么对待 |
|---|---|---|
| A级 | 公告/财报/交易所，最硬 | 可信 |
| B级 | 券商研报 | 较可信，是研报观点 |
| C级 | 专家会议/媒体 | 参考，需验证 |
| D级 | 群聊/小作文 | 只当线索，别当依据 |
| ·待验证 | 还没用真实数据坐实 | 别单独拿来做决定 |

**"数字校验 N"**：这条结论里有 N 个数字回原文核对过、真实存在。越多越靠谱。

## 六、能做什么、不能做什么（实话实说）

**能做** ✅
- 把各类信息抽成带成色、带数字校验、可溯源的知识库
- **批判性质疑**：自动揪出过于乐观的数字（同类分位对照）、没出处的猜测、回测软肋
- **联网采信权威信源**：公告/投行/权威媒体核实，自媒体不采信（需开 `TKB_USE_WEB=1` 接源）
- 量化因子研报问答（你的 63 篇主力是这类，问因子最顺手）
- 时序追踪：被证实/证伪自动更新

**暂时弱/不能** ⚠️
- **联网真实抓取**：巨潮公告抓取**已实测接通**（`announcement.py`，沪深全量+上交所兜底+退避重试），默认关闭，`TKB_USE_WEB=1` 启用
- **产业链拆解**：需行业研报才有料；现库 62/63 是量化研报，这块暂空，灌行业研报后激活
- **逻辑漏洞深挖**：乐观/软肋的"规则版"已有；更深的逻辑质疑（幸存者偏差等）需接 LLM
- **自动交易**：只给研究素材，不下单、不荐股

## 七、出问题怎么办

| 现象 | 处理 |
|---|---|
| `command not found: ./tkb` | 确认在 `~/trading_kb` 下；或 `chmod +x tkb` |
| 返回"证据不足" | 库里没这主题，换词或先入相关研报 |
| 想从头重来 | 删 `~/trading_kb/data/` 里的 `.db`，再 `./tkb ingest` |
| 确认系统正常 | `PYTHONPATH=src python3 -m pytest -q tests/` 应全过 |

## 八、进阶开关（普通用户可忽略）
