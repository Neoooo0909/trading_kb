# trading_kb 使用说明（普通用户版）

> 一份不需要懂代码也能上手的操作指南。照着做即可。

---

## 一、这个东西能帮我做什么？

把它想成你的**研报智能管家**。你把看过的研报丢给它，它会自动：

- 📥 **读研报**：把研报里的关键结论、数字、因子全部抽出来，存进知识库
- ✅ **标可信度**：每条结论标上"成色"（公告级 / 研报级 / 专家级 / 群聊级），帮你分清"硬证据"和"小作文"
- 🔢 **查数字**：研报里的每个数字都回原文核对过，防止 AI 编数字
- 🔍 **问答**：你问"某某因子表现怎么样"，它给你一份带证据、带出处的结构化回答
- 📊 **追踪变化**：同一件事后来被证实还是被推翻，它都记着，不健忘

**一句话**：让你看过的研报"存得下、查得到、信得过"。

---

## 二、开始之前（只需确认一次）

这套系统站在你已有的 **report_lab**（研报处理工具）肩膀上。两者关系：

```
你的研报 PDF  →  report_lab（读PDF、抽卡片、校验数字）  →  trading_kb（建知识库、问答）
```

确认两点：
1. 电脑装了 Python 3（终端输入 `python3 --version` 能看到版本号即可）
2. `~/report_lab/` 里已经有处理好的研报（目前已有 63 篇，可直接用）

> 不用安装任何东西。系统只用 Python 自带的功能，开箱即用。

## 三、最简上手：3 条命令

打开"终端"（Terminal），先进入项目目录：

```bash
cd ~/trading_kb
```

之后所有命令都用 `./tkb` 开头。三条核心命令：

### 1️⃣ 把研报装进知识库

```bash
./tkb ingest
```

它会读取 report_lab 里的所有研报，自动分类、标成色、入库。跑完会显示一份回执：

```
卡片 63 | findings 1562 | 实体登记 906
硬事实 5 | 量化事实 1370 | 结构关系 0 | 背景(跳过) 187
成色分布 {'A': 0, 'B': 1371, 'C': 4, 'D': 0}
```

> 读法：处理了 63 篇研报、抽出 1562 条结论，其中 1370 条是量化因子结论。成色大多是 B 级（研报级）。
> **可以反复跑，不会重复入库**（系统会自动去重）。

### 2️⃣ 提问

```bash
./tkb ask "多空因子 选股效果"
```

它会给你一份"六段式"回答（下一节教你看懂）。

### 3️⃣ 看知识库有多大

```bash
./tkb stats
```

## 四、怎么看懂回答（重要）

每次提问，回答固定分**六段**，照着读就行：

```
## 结论          ← 一句话核心答案，带成色标签
## 证据链        ← 支撑结论的多条证据，每条带成色+出处
## 分歧/反证      ← 有没有相反观点 / 被推翻的旧说法
## 后续验证       ← 哪些还没坐实、需要你盯着的
## 交易含义       ← 对交易的提示（低成色的别当买点）
## 引用来源       ← 每条结论来自哪篇研报，可回查
```

### 看懂"成色标签"

回答里每条结论后面有个标签，比如 `[B级·待验证]`：

| 成色 | 含义 | 怎么对待 |
|---|---|---|
| **A级** | 公告/财报/交易所，**最硬** | 可信 |
| **B级** | 券商研报 | 较可信，但是研报观点 |
| **C级** | 专家会议/媒体 | 参考，需验证 |
| **D级** | 群聊/小作文 | **只当线索，别当依据** |
| `·待验证` | 还没用真实数据坐实 | 别单独拿来做决定 |

### 看懂"数字校验 N"

证据链里每条后面有 `数字校验3` 这种标记——意思是这条结论里有 **3 个数字回原文核对过、真实存在**。数字越多核对过，越靠谱。

### 真实例子

```bash
./tkb ask "高频因子 低频化"
```
```
## 结论
高频因子低频化系列整体表现出色，"暗流涌动"因子多空收益24.73%… [B级·待验证]

## 证据链
[F1] [B级·待验证] …多空收益24.73%…  (来源1篇, 数字校验2)
[F2] [B级·待验证] 滴水穿石因子与现有高频因子相关性较低。 (来源1篇, 数字校验0)
...
```
读法：这是券商研报观点（B级），核心结论里 2 个数字核对过。想深究就看"引用来源"段去翻原研报。

## 五、常见任务怎么做

| 我想…… | 这样做 |
|---|---|
| 查某个因子表现 | `./tkb ask "动量因子 表现"` |
| 查某主题/概念 | `./tkb ask "换手率 因子"`、`./tkb ask "沪深300 增强"` |
| 查某只股票的逻辑 | `./tkb ask "绿的谐波 定点"`（库里没有会提示"证据不足"）|
| 看历史/被推翻的说法 | `./tkb ask "某查询" --audit`（多显示已证伪/被替代的旧事实）|
| 新增研报后更新库 | 先用 report_lab 处理新研报，再 `./tkb ingest` |
| 看舆情功能演示 | `./tkb sentiment-demo` |

> **提示**：查询词用空格分开几个关键词效果最好，比如 `"多空 因子 选股"`。不加空格也能搜，但分开更准。

## 六、怎么加新研报进来？

知识库的"原料"来自 report_lab。加一篇新研报的完整流程：

```bash
# 第一步：用 report_lab 处理新研报（读PDF、抽卡片、校验数字）
cd ~/report_lab/scripts
python3 ingest.py ~/Downloads/某某研报.pdf
python3 extract.py
python3 verify.py

# 第二步：让 trading_kb 把新研报纳入知识库
cd ~/trading_kb
./tkb ingest
```

之后就能 `./tkb ask` 问到新研报的内容了。

## 七、聊天记录/短评怎么处理？

聊天记录、群消息、短评这类"碎片"**走单独的轻通道**，和研报分开（避免噪声淹没正经证据）。系统会：

- 只保留提到你关注标的的碎片，其余冷存
- 默认标成 **D级（群聊）**，隔离起来，不污染研报证据
- 把碎片聚合成"情绪/热度信号"，而不是单条乱翻
- 只有被研报/公告印证后，才"升级"成正经证据

想看它怎么工作：
```bash
./tkb sentiment-demo
```

> 目前这是演示功能；要接你自己的聊天数据，需要进一步配置（可让维护者帮你接）。

## 八、当前能做什么、不能做什么（实话实说）

**能做** ✅
- 把研报抽成带成色、带数字校验、可溯源的知识库
- 量化因子研报的问答（你现在的 63 篇主要是这类，问因子最顺手）
- 时序追踪：同一件事被证实/证伪会自动更新状态

**暂时弱/不能** ⚠️
- **产业链拆解**：需要"行业研报"才能建产业链图；你现在的库 62/63 是量化研报，所以这块暂时是空的。灌入行业/公司研报后会自动激活。
- **"最高/最低"这类排序问题**：它能找到相关结论，但还不理解"哪个 IC 最高"这种数值排序，需要你自己在证据链里看。
- **自动交易**：它只给研究素材，**不替你下单、不荐股**。

## 九、出问题怎么办

| 现象 | 处理 |
|---|---|
| `command not found: ./tkb` | 确认在 `~/trading_kb` 目录下；或运行 `chmod +x tkb` |
| 提问返回"证据不足" | 库里没这个主题的研报，换个词，或先入相关研报 |
| 想从头重来 | 删除 `~/trading_kb/data/` 里的 `.db` 文件，再 `./tkb ingest` |
| 想确认系统正常 | `PYTHONPATH=src python3 -m pytest -q tests/` 应显示全部通过 |

## 十、名词小词典

- **研报重 lane / 舆情轻 lane**：两条处理通道——研报走"重质量"通道，聊天碎片走"轻舆情"通道。
- **成色（A/B/C/D）**：证据可信度，公告 > 研报 > 专家会 > 群聊。
- **待验证（unverifiable）**：还没用真实数据坐实的结论。
- **事实/findings**：从研报里抽出的一条条结论。
- **去重**：同一条结论来自多篇研报时，合并成一条并累加"被几篇支持"。
- **证伪不删除**：某结论被推翻后，系统不删它，而是标记"已证伪"，可用 `--audit` 查回——方便复盘。

## 十一、进阶开关（普通用户可忽略）

| 想要 | 设置 | 说明 |
|---|---|---|
| 用真实数据验成色 | `TKB_USE_DATA_VERIFY=1 ./tkb ingest` | 会查公告/财务，消耗 iFinD 额度 |
| 启用 LLM 分类 | `TKB_USE_LLM=1`（需先接好 LLM 钩子）| 默认走规则，更快可复现 |
| 换数据存放目录 | `TKB_DATA_DIR=/某路径 ./tkb stats` | 默认 `~/trading_kb/data` |

---

**就这些。日常你只需要记住三条：`./tkb ingest`（入库）、`./tkb ask "问题"`（提问）、`./tkb stats`（看规模）。**
